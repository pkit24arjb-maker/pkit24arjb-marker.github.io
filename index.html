What is Full Stack? HTML CSS Bootstrap JavaScript

1) What is Full Stack Development?

My understanding: Full stack development means building both the front‑end (what users see and interact with in the browser) and the back‑end (server logic and database) so everything works together. A typical request flows like this: the browser sends a request → the server handles it and talks to the database → a response (usually HTML/JSON) comes back to the browser.

Common stacks include MERN (MongoDB, Express, React, Node) and LAMP (Linux, Apache, MySQL, PHP). Even when I’m only showing front‑end examples here, I understand how the back‑end would expose APIs the front‑end can call.

If I was extending this, I’d add a tiny Node/Express API and fetch it from my JavaScript example.

2) HTML

HTML gives a page its structure using semantic tags like <header>, <main>, and <footer>. I also make sure to include alt text on images for accessibility and use headings in order (H1 → H2 → H3).

  • Document outline: <!DOCTYPE html>, language, and meta tags
  • Links, images with alt, and lists
  • Semantic layout improves accessibility and SEO

3) CSS

CSS controls presentation (colours, spacing, layout). I used an external stylesheet to keep styles separate from HTML. I also used the cascade and classes to style repeated elements.

  • Typography and colour variables
  • Responsive layout with simple media queries
  • Reusable utility classes

4) Bootstrap Framework

Bootstrap provides a responsive grid and pre‑built components (navbars, cards, buttons). I used the CDN version to quickly add it, then used the grid to build a layout that collapses nicely on mobile.

  • Grid classes like .container, .row, .col-md-6
  • Navbar, cards, buttons
  • Utilities (e.g., spacing, text alignment)

JavaScript makes pages interactive by manipulating the DOM and handling events. In my example I added a button that updates content, a live clock, and a simple form validator.

  • Event listeners and DOM updates
  • Date/time and timers
